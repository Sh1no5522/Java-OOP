1. When to use an Interface vs when to use an abstract class[cite: 67, 68].
- Use an Abstract Class when you want to share code among several closely related classes, or when you expect classes that extend your abstract class to have many common methods or fields.
- Use an Interface when you expect that unrelated classes would implement your interface (e.g., Comparable or Cloneable). Interfaces are contracts for what a class can do, without dictating how it does it.
